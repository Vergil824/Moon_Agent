# **产品需求文档 (PRD) - 满月 Moon (内衣推荐智能 Agent)**

## **1. 文档信息 (Document Information)**

- **项目名称**: 满月 Moon
- **版本**: v1.1
- **状态**: 进行中 (In Progress)
- **更新日期**: 2025-12-25
- **主要更新**: 增加用户系统 (WeChat Login)、电商闭环 (Cart & Payment)、售后流程。

## **2. 项目背景与目标 (Project Background & Objectives)**

### **2.1 项目背景**

许多女性在选购内衣时面临尺码不准、胸型认知错误、穿着不适（跑杯、空杯、勒痕）等痛点。现有的电商搜索缺乏个性化指导，而线下导购往往带有销售压力。我们需要一个像“贴心闺蜜”一样的 AI Agent，通过自然的对话引导用户了解自己的身体，并推荐真正合适的内衣。

### **2.2 核心目标**

- **短期目标 (2025.12 月)**:
  - 实现完整闭环：从智能对话诊断到购物车结算，再到支付完成的 H5 全链路。
  - 集成微信登录，打通用户身体数据与订单数据的关联。
- **长期价值**: 建立用户对“满月”的信任感，解决内衣选购痛点，实现高匹配度的商品转化，降低内衣行业的退货率。

## **3. 用户画像与人设 (Personas)**

### **3.1 目标用户 (Target User)**

- **特征**: 曾有过内衣购买失败经历，对自己的尺码/胸型不确定，希望获得专业建议但排斥说教式推销。
- **核心需求**: 舒适、合身、被理解、解决具体痛点（如副乳、空杯）。

### **3.2 Agent 人设 - 满月 (Moon)**

- **定位**: 贴心大胸闺蜜，内衣选购专家。
- **性格**: 热情、直爽、专业但不枯燥、共情能力强。
- **口头禅**: “来啦！”，“各过各的”，“差不多”，“为什么？”。
- **沟通风格**: 使用“宝宝/姐妹”等亲昵称呼，拒绝机械客服腔，使用表情包和生动的比喻。

## **4. 功能需求 (Functional Requirements)**

### **4.1 用户与权限系统 (User & Auth)**

- **微信一键登录**: 支持微信公众号/H5 静默授权登录。
- **用户画像同步**: 登录后自动关联用户的测量记录 (Measurements) 和诊断结果。

### **4.2 核心对话流程 (Core Conversation Flow)**

#### **节点一：欢迎与破冰**

- **功能**: 确立人设，建立连接。使用全新的 ChatHeader 和 UI 布局。

#### **节点二：身体数据采集 (尺码)**

- **交互**: 引导测量。配图：3D 灰模演示拉紧软尺效果。输入下胸围 -> 输入上胸围。

#### **节点三：辅助信息采集**

- **交互**: 弹出水平滑块 (Slider)，收集 **身高、体重、腰围**。

#### **节点四：胸型诊断**

- **选项**: 圆盘型 (散)、纺锤型、半球型。使用 3D 抽象图标引导。

#### **节点五：痛点诊断**

- **交互**: 多选网格 (Grid Layout)。包括：隐形刺客、逃跑新娘、分身术、空城计、滑坠深坑。

#### **节点六：分析与加载**

- **交互**: 进度条反馈，鼓励性文案。

### **4.3 电商闭环 (Ecommerce Flow)**

#### **商品推荐 (Recommendation)**

- **逻辑**: 基于诊断出的尺码和胸型，动态生成 SQL 从 Supabase 过滤商品。
- **展示**: 推荐理由标签（如“因为你是圆盘型...”），突出服务保障（7 天无理由退换）。

#### **购物车 (Shopping Cart)**

- **功能**: 商品规格选择 (SKU)、数量加减、选中/全选、合计金额。
- **交互**: 底部常驻 BottomNav 可随时进入。

#### **支付结算 (Payment)**

- **渠道**: 优先支持 **支付宝 (Alipay)**。
- **流程**: 购物车结算 -> 订单创建 -> 唤起支付 -> 支付成功反馈。

#### **个人中心 (User Center)**

- **功能**: 查看订单列表、售后申请、收货地址管理。

### **4.4 售后服务 (After-Sales)**

- **服务保障**: 所有推荐商品必须支持 7 天无理由退换。
- **退换货流程**: 用户可在订单列表发起退换货，上传凭证，跟踪售后状态。

## **5. 非功能需求 (Non-Functional Requirements)**

- **隐私保护**: 身体数据加密存储，仅用于推荐。
- **响应速度**: 对话响应 < 2s，支付跳转流畅。
- **移动端适配**: 重点优化微信 H5 环境。

## **6. 成功指标 (Success Metrics)**

- **转化率 (CVR)**: 从进入对话到完成支付的用户比例。
- **推荐准确度**: 用户反馈“合身”的评价比例。
- **退货率**: 通过 Agent 推荐购买的退货率低于传统电商平均水平。
