# UX设计与UI规范：内衣专家Agent ("撑撑姐")

**作者:** Sally (UX Designer)
**日期:** 2025年12月12日
**版本:** 2.0 (基于Figma原型迭代)
**来源:** PRD.md, Figma Prototype
**状态:** 准备开发

## 1. 设计哲学

**“支持性镜像” (The Supportive Mirror)**
我们的核心体验是建立信任。“撑撑姐”不是来评判你的身材，而是来利用数据帮助你理解它。整个 UI 流程必须像一场深度的、私密的但又非常专业的咨询对话。

- **核心情感:** 专业信赖、释然、被理解。
- **视觉隐喻:**
    - **Agent:** 温暖、智慧的引导者（通过对话气泡体现）。
    - **数据/模型:** 冷静、客观、医学/建筑感（通过灰色 3D 模型体现）。
- **交互节奏:** 对话驱动。用户每输入一个数据点，Agent 都会给予反馈，确保流程感觉像是在“合作”而不是单纯的“填表”。

## 2. 设计系统 (更新)

基于 Figma 原型 的视觉风格进行调整。

### 2.1 配色方案

- **主色调 (品牌/交互):** `紫罗兰渐变 (Violet Gradient)` (例如：#A855F7 到 #7C3AED) - 用于主要行动按钮 (CTA)、选中状态、进度条和强调文本。紫色传达了智慧、尊贵和一点点神秘科技感。
- **辅助色 (数据/客观):** `板岩蓝/冷灰 (Slate Blue/Cool Grey)` (#475569) - 用于测量数值、医学标注线条和次要文本。
- **背景色:** `纯净白 (Clean White)` (#FFFFFF) 或极浅的灰白，保持界面干净、现代。
- **3D模型材质:** `哑光建筑灰 (Matte Architectural Grey)` (#E2E8F0) - 强调结构而非肉体感。

### 2.2 排版与组件风格

- **字体:** 保持清洁、现代的无衬线字体（如 Inter 或 System UI）。
- **卡片式设计:** 内容被封装在圆角卡片中，漂浮在背景之上，营造层次感。
- **对话流:** 界面核心是聊天流。用户的操作区域通常位于屏幕底部或作为对话的一部分出现。

## 3. 用户流程与界面规范 (基于原型)

### 节点 1: 数据输入与引导 (Measurement & BMI)

**目标:** 降低阻力，提供清晰的测量指导。

- **UI 结构:** 对话气泡引导 + 底部操作卡片。
- **测量输入 (Measurement Screens):**
    - **交互:** 简单的数字输入框 (cm)。
    - **指导辅助:** 关键在于 "Demo1" 和 "Demo2" 屏幕。当用户需要测量时，提供清晰的 **3D 动画循环**，展示正确的测量姿势（如：如何拉紧软尺测下围，如何 90度弯腰测上围）。
- **身体数据 (BMI Screen):**
    - **交互:** 使用流畅的水平滑块 (Sliders) 来输入身高 (cm) 和体重 (kg)。滑块控件使用主色调紫色。

### 节点 2: 自我认知探索 (Types & Pain Points)

**目标:** 帮助用户定性描述自己的身体特征和困扰。

- **胸型选择 (Types Screens):**
    - **UI 组件:** 可垂直滚动的选项卡片列表。
    - **卡片设计:** 左侧为抽象的、圆润的 **3D 线条图标**（非写实照片），右侧为标题（如“圆盘型”）和简短描述。
    - **选中状态:** 点击卡片后，卡片边框和图标高亮显示为紫色，并出现勾选标记。
- **痛点诊断 (Pain Points Screens):**
    - **UI 组件:** 多选网格 (Grid Layout)。
    - **卡片设计:** 每个选项是一个带有图标和简短文字的小方块（如：带闪电图标的“肩带滑落”）。
    - **交互:** 用户点击选择多个痛点。选中项背景变为浅紫色，边框高亮，表明 Agent 已记录下这个抱怨。

### 节点 3: 分析与揭晓 (Analysis & Result)

**目标:** 营造期待感，然后提供高价值的“Aha!”时刻。

- **分析过渡 (Analysing Screen):**
    - **视觉:** 专门的加载屏幕。显示进度条或加载动画。
    - **文案:** “撑撑姐”的安抚性话术，例如：“正在综合你的数据...”，让等待过程不枯燥。
- **3D 身体蓝图 (3D Model Screen):**
    - **核心体验:** 这是整个流程的高光时刻。
    - **视觉:** 屏幕中央展示用户专属的 **灰色哑光 3D 人台模型**。
    - **数据可视化:** 使用冷色调的线条和标签，直接在模型上标注关键分析结果（例如：指出“底盘宽”、“高鸡心”的具体位置）。这比纯文字报告更直观、更具说服力。
- **诊断报告 (Report Screen):**
    - **UI 组件:** 一个总结性的数据卡片。
    - **关键对比:** 明确展示“之前的错误认知”（划掉的旧尺码）与“现在的真实尺码”（醒目的新尺码，配以绿色/紫色强调）。
    - **详细数据:** 列出所有测量维度的最终数字，作为客观依据。

### 节点 4: 解决方案 (Recommendations)

**目标:** 将诊断转化为行动。

- **推荐列表 (Recommand... Screen):**
    - **UI 组件:** 产品推荐卡片流。
    - **卡片内容:** 产品图片、名称、以及最重要的——**推荐理由**。理由必须回扣之前的诊断（例如：“因为你是[圆盘型]，这款的[大钢圈]能提供更好包裹”）。

## 4. 原型所需的关键视觉资产 (更新列表)

为了完美实现 Figma 原型中的效果，我们需要准备以下资产：

1.  **3D 测量引导动画:**
    * 直立测量下胸围的循环动画 (Demo1)。
    * 90度弯腰测量上胸围的循环动画 (Demo2)。
2.  **抽象 3D 胸型图标:** 一套风格统一、圆润、非写实的 3D 图标，用于表示圆盘型、纺锤型、半球型等。
3.  **痛点图标集:** 一套扁平或微立体的图标，用于表示空杯、压胸、肩带滑落等问题。
4.  **参数化 3D 人台模型:** 一个基础的灰色人台模型，能够根据输入参数（胸围差、底盘宽窄等）动态调整形态，并支持在特定部位添加标注点。
